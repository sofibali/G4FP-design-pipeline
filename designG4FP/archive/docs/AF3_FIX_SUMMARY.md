# AlphaFold3 Prediction Issue - RESOLVED

## Problem

AlphaFold3 predictions were failing with this error:
```
ValueError: AlphaFold 3 input JSON contains sequences with unset IDs.
```

The directories were created but no model files (`*_model.cif`) were generated because AF3 failed immediately after reading the input JSON files.

## Root Cause

The JSON input files generated by the pipeline were **missing the required `id` field** for each sequence in the `sequences` array.

### Before (incorrect):
```json
{
  "sequences": [
    {
      "protein": {
        "sequence": "RSATM..."
      }
    }
  ]
}
```

### After (correct):
```json
{
  "sequences": [
    {
      "protein": {
        "sequence": "RSATM..."
      },
      "id": "1"
    }
  ]
}
```

## Solution Applied

Created and ran `fix_af3_json_ids.py` to add sequential `id` fields to all sequences in all JSON files.

**Results:**
- ✅ Fixed: **800 JSON files** across 4 output directories
- ✅ Each sequence now has a unique `id` within its input file
- ✅ Both bound and apo inputs fixed

## Prevention

To prevent this issue in future pipeline runs, the JSON generation script should be updated to include `id` fields when creating sequences. The fix would be in the script that generates the AlphaFold3 input JSONs (likely `02_generate_alphafold3_inputs.py` or similar).

### Recommended fix for generator script:

```python
# When building sequences array
sequences = [
    {
        "protein": {"sequence": seq},
        "id": "1"  # <-- Add this
    }
]

# For bound state with ligands:
sequences = [
    {
        "protein": {"sequence": seq},
        "id": "1"  # <-- Add this
    },
    {
        "dna": {"sequence": "GGGTGGGTGGGTGGGT"},
        "id": "2"  # <-- Add this
    },
    {
        "type": "K",
        "count": 3,
        "id": "3"  # <-- Add this
    }
]
```
