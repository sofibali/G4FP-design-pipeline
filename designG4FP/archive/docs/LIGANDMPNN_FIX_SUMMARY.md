# LigandMPNN Issue: All Sequences Identical

## Problem
All 10,000 sequences generated by LigandMPNN were identical with confidence scores of 1.0.

## Root Cause
**LigandMPNN was not designing any residues (`num_res=0`)**

The config file specified: `design_residues: A15-60 A240-285`

However, LigandMPNN's `--redesigned_residues` parameter requires:
- **Space-separated individual residues**: `A15 A16 A17 ... A60 A240 A241 ...`
- **NOT range notation**: `A15-60`

## Fix Applied
Updated [`01_run_ligandmpnn.sh`](01_run_ligandmpnn.sh) to:

1. **Expand residue ranges automatically**:
   ```python
   def expand_residues(residue_spec):
       # Converts "A15-60 A240-285" to "A15 A16 A17 ... A285"
       parts = residue_spec.strip().split()
       expanded = []
       for part in parts:
           match = re.match(r'([A-Z]+)(\d+)-(\d+)', part)
           if match:
               chain, start, end = match.groups()
               expanded.extend([f'{chain}{i}' for i in range(int(start), int(end) + 1)])
           else:
               expanded.append(part)
       return ' '.join(expanded)
   ```

2. **Activate correct conda environment**:
   ```bash
   eval "$(conda shell.bash hook)"
   conda activate unified_mpnn
   ```
